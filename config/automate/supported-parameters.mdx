## Overview

WiCAN vehicle profiles use named parameters to identify what each PID expression computes. These names have two purposes:

1. They appear as attribute names in MQTT output and HTTP JSON responses.
2. They are recognized by the [Home Assistant integration](/home-assistant/integration-setup), which uses the `class` and `unit` metadata to auto-configure entities with the correct device class, unit of measurement, and display behavior.

When writing a [custom PID](/config/automate/usage#custom-pids) or [vehicle profile](/config/automate/vehicle-profiles), use a name from the table below where a matching parameter exists. This ensures correct integration with Home Assistant and consistent output across the ecosystem.

<Info>
For standard OBD-II parameters available without a vehicle profile, see [Standard PIDs](/config/automate/usage#standard-pids) and use the **Scan PIDs** button in the Automate tab.
</Info>

## How parameters map to expressions

Each parameter is computed from the raw bytes returned by an ECU PID request. The `expression` field in a vehicle profile or custom PID defines how to map those bytes to the final value.

For example, to extract State of Charge from a two-byte response:

```json
{
  "name": "SOC",
  "expression": "[B0:B1] / 10",
  "unit": "%",
  "class": "battery",
  "min": "0",
  "max": "100"
}
```

Here `[B0:B1]` reads bytes 0 and 1 as a 16-bit unsigned integer, then divides by 10 to get a percentage. See the [expression format reference](/config/automate/usage#expression-format) for the full syntax.

### Common expression patterns by category

| Category | Expression | Example parameter |
|---|---|---|
| Direct byte value | `B0` | `SPEED` (km/h) |
| Temperature offset | `B0 - 40` | `COOLANT_TMP`, `BATT_TEMP` |
| Percentage (0‚Äì255 scale) | `B0 * 100 / 255` | `FUEL`, `THROTTLE` |
| 16-bit integer √∑ 10 | `[B0:B1] / 10` | `SOC`, `HV_V`, `HV_A` |
| 16-bit integer √∑ 4 | `[B0:B1] / 4` | `ENGINE_RPM` |
| 16-bit millivolts to V | `[B0:B1] / 1000` | `LV_V` |
| 32-bit integer | `[B0:B3]` | `ODOMETER` |
| Signed current encoding | `(65536 - [B0:B1]) / 10` | `HV_A` (some EVs) |
